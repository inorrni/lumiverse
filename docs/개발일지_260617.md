# 개발일지 — 2026.06.17

> [!note] 한 줄 요약
> 온보딩(모드→목표→경로설계)에서 **죽은 UI·거짓 라벨·뒤로가기 입력 손실**을 정리했다. "알아서" 모드에선 무의미하던 강도(살살/스파르타) 선택을 추천 모드에서만 노출하고, 분해/직접입력을 아우르도록 **`/breakdown`→`/plan`** 으로 개명했으며, 화면 간 router state를 **양방향**으로 흘려 뒤로가기 시 목표·디데이가 유지되게 했다.

## 3. 변경 내역

### 3-1. 모드 선택 — 조건부 노출 (feat)
- 강도(살살/스파르타)는 **AI 분해 프롬프트(`intensity`)에만** 영향을 준다. "알아서" 모드는 AI를 호출하지 않으므로 강도 값이 **어디에도 쓰이지 않는 죽은 UI**였다.
- 순서를 뒤집어 **입력방식(알아서/추천) 먼저** 보여주고, `inputMethod === 'recommend'`일 때만 강도 카드를 펼친다.
- 기본 프리선택(추천·살살) 유지로 스킵 방지.

### 3-2. 진행 버튼 라벨 분기 (feat)
- 기존엔 "알아서" 모드에서도 버튼이 **"우주 항해사에게 분해받기"** 로 떠 실제 동작(빈 직접입력 화면)과 어긋났다(거짓 라벨).
- 분기: 추천=`우주 항해사에게 분해받기 ▸` / 알아서=**`내 우주 직접 그리기 ▸`**.

### 3-3. `/breakdown` → `/plan` 개명 (refactor)
- 해당 화면은 추천=AI 분해 / 알아서=빈 목록 직접입력을 **한 화면에서** 처리(에디터 공유)한다. 그런데 이름이 "breakdown"이라 직접입력 흐름과 의미가 어긋났다.
- 라우트 `/breakdown`→`/plan`, 컴포넌트 `AiBreakdownPage`→`PlanPage`(+`.module.css`, 로컬 `BreakdownLoading`→`PlanLoading`). 파일은 `git mv`로 히스토리 보존.
- **유지한 것**: `lib/api.js`의 `breakdownGoal`(실제 Supabase `goal-breakdown` Edge Function 호출 — 진짜 분해 API), `src/sample/`(디자인 캔버스 목업).

### 3-4. 뒤로가기 입력 복원 (fix)
- 원인: 이 페이지들은 부모-자식이 아니라 **라우트 형제**라 props가 아닌 **`location.state`** 로 값을 주고받는다. 전진(`/goal→/plan`)은 state를 넘기지만, `BackRow`가 `to`만 지원해 **뒤로갈 땐 state 없이** 이동 → `GoalInputPage`가 빈값으로 재마운트.
- 조치: `BackRow`에 선택적 `state` prop 추가(`navigate(to, { state })`) → `PlanPage`가 받은 state를 뒤로 그대로 전달 → `GoalInputPage`가 `useState(state?.goal || '')`로 초깃값 복원.
- 함께 정리: 추천 분해 결과가 빈 EmptyView의 **"목표 다시 입력"** 버튼이 mode/inputMethod까지 잃던 것을 강도·입력방식은 유지하도록 보정.

### 3-5. D-Day 캘린더 아이콘 제거 (chore)
- `type="date"` 입력은 브라우저 기본 달력 아이콘을 이미 띄워 커스텀 SVG가 중복 → `CalendarIcon` 컴포넌트·`right` 슬롯 사용 제거. (`TextInput`의 `right` 슬롯 자체는 글자수 카운터에서 계속 사용)

## 8. 입력칸 포커스 테두리 — `.input` → `.wrap` 전체 (fix)

- 원인: 전역 `base.css`의 `:focus-visible { outline: 2px solid var(--accent) }` 포커스 링이 내부 `<input>`에 걸림. (`.wrap:focus-within` 테두리 변화는 `--line`→`--text-mid`라 링에 묻힘)

## 9. '다른 분해 추천받기' 리프레시 (feat · 프론트+백엔드)

## 14. 사후분석 — goal-breakdown BOOT_ERROR (변수명 충돌)
> [!danger] 증상
> localhost에서 `POST .../goal-breakdown` CORS 에러: "preflight ... does not have HTTP ok status". OPTIONS 직접 호출 시 **HTTP 503 + `sb-error-code: BOOT_ERROR`** — 함수가 부팅 자체에 실패해 모든 요청(OPTIONS 포함) 503.

- **근본 원인**: count 기능 추가 시 `const count`(개수 정규화)를 선언했는데, 약 20줄 아래 rate limit의 기존 `const { count }`와 **같은 스코프 중복 선언** → `SyntaxError: Identifier 'count' has already been declared` → 모듈 로드 실패.

# 개발일지 (후속) — 2026.06.17 · 7차

> [!note] 한 줄 요약
> 하단 "점검" 탭을 per-goal 중간점검 리다이렉트에서 **내 우주 전체를 읽는 자기 인사이트 페이지**로 전환했다. 모든 목표 통계 + 한줄평 corpus를 `self-insight` Edge Function에 주입해 **전체 상태·잘 하는 키워드·자주 하는 말·피하고 싶은 것·항해사 한마디**를 받아 **읽기 전용**으로 안내한다(목표 수정 아님). per-goal 중간점검(`/app/check/:id`)은 그대로. 별도 브랜치 `feat/universe-insight`.

## 31. 점검 탭 → 우주 전체 자기 인사이트 (feat)
- 라우팅: `/app/check` → `UniverseCheckPage`(기존 `MidCheckIndex` 리다이렉트 제거). `/app/check/:id`(목표별 중간점검)는 유지.
- 집계(클라): `useGoals().goals` + `historyOf`로 **목표별 전체 내용**(이름·강도·`computeMidCheck` 상태·완료율·세부목표 목록·그 목표의 한줄평)으로 묶어 전달 → LLM이 **모든 목표를 가로질러 종합**. + 합계·평균 진행률.
- 화면(읽기 전용): 헤더(목표 수·별·평균 진행률) → 전체 상태 카드 → 잘 진행 중/자주 하는 말/피하고 싶은 것 **칩**(비면 섹션 숨김) → 항해사 말풍선. 목표 0개 빈 상태 / 로딩 / 실패 폴백+재시도.

## 32. `self-insight` Edge Function (feat · 백엔드)
- `mid-check` 패턴 미러(JWT·`lumiverse_llm_cache`·rate limit·`chatJSON`, trigger=`self_insight`).
- 입력은 **목표 단위 구조**(세부목표+한줄평 포함). 프롬프트가 **단일 목표 요약 금지 → 목표 교차 종합으로 "당신"을 판단**하도록 강제, 출력 어투도 **2인칭("당신은 …")**(3인칭 "사용자" 금지).
- 출력 `{ summary, doing_well[], frequent_words[], dislikes[], message }`. 키워드/단어/싫어하는 것은 **목표·세부목표·한줄평의 실제 근거로만** 추출, 없으면 빈 배열(지어내지 않음). 이모지·과한 쾌활함 금지. 캐시 키 = 목표별 내용 해시. `deno check` 통과.
- 인사이트 결과는 **DB 미저장**(파생·휘발, 데이터 바뀌면 재계산).
