# 개발일지 — 2026.06.20

> [!note] 한 줄 요약
> 블랙홀(보관함) 관련 결함 3건을 근본 원인까지 짚어 수정했다. ① **보완으로 추가한 행성이 대시보드 Today 카드에 안 보임**(카드가 `steps.slice(0,3)`로 앞 3개만 렌더 → 맨 뒤에 붙는 새 행성 잘림), ② **포기(마무리)한 목표 은하가 블랙홀에 안 보이고 복구 경로도 없음**(보관함이 행성만 조회), ③ 행성 복구 후에도 새 행성이 Today에 안 보임(①과 동일 원인). 브랜치 `claude/black-hole-feature-bugs-skwbrk`.

## 1. 배경 — 무엇이 안 됐나

블랙홀은 0618에 "보완으로 보낸 행성을 되살리는 안전망"으로 들어왔고(이슈 J1), 같은 평가보고서에서 *목표 단위 블랙홀(마무리한 은하)은 보관함에 미노출(행성 전용)* 이라고 이미 한계로 적어 두었다. 이번에 사용자 점검에서 다음 3가지가 보고됐다.

1. 중간점검 → **보완하기**로 기존 행성을 블랙홀에 보내고 추천 행성을 추가한 뒤, 메인화면 **Today 칸에 새로 담은 행성이 나타나지 않음**.
2. **목표 은하를 포기**한 뒤 블랙홀에 가면 그 **은하가 보이지 않음**.
3. 블랙홀에서 **행성 복구**를 해도 1에서 추가한 새 행성이 Today에 안 보임.

## 2. 근본 원인

> [!important] 핵심
> 1·3은 **데이터가 아니라 표시(View) 문제**였다. 새 행성은 `addPlanet`이 오늘~디데이 별과 함께 정상 생성하며 우주맵·"모두 보기"(TodayPage)에는 보였다. 2는 **은하·행성을 다른 상태 축으로 보관**하는데 보관함이 행성만 봤기 때문이다.

- **버그 1·3 (Today 카드)** — [DashboardPage.jsx](../src/pages/DashboardPage.jsx)
  - Today 카드가 `activeGoal.steps.slice(0, 3)`로 **앞 3개 행성만** 렌더했다. 새 행성은 `order_idx`가 가장 커서 정렬상 맨 뒤 → 행성이 3개 이상이면 잘려 안 보임.
  - 게다가 카드 목록은 `todayStarId` 필터 없이 전체 step을 slice 했는데, 헤더 카운트(`todayTotal`)는 `todayStarId` 기준이라 **목록과 카운트가 불일치**할 수 있었다.
- **버그 2 (포기 은하 미노출)** — [GoalStore.jsx](../src/store/GoalStore.jsx) · [BlackHolePage.jsx](../src/pages/BlackHolePage.jsx)
  - `removeGoal`은 **은하**의 `status='blackhole'`만 토글한다(소속 행성은 `active` 유지).
  - 반면 `BlackHolePage`는 `loadBlackholePlanets`로 **행성만** 조회하고, 그 쿼리는 `galaxy.status === 'active'`인 행성만 통과시킨다. 결국 포기한 은하는 (행성도 active, 은하는 blackhole이라) **어디에도 안 나오고 복구 수단도 없었다**.

## 3. 적용 지점

| 파일 | 변경 |
|---|---|
| [DashboardPage.jsx](../src/pages/DashboardPage.jsx) | `todaySteps = activeGoal.steps.filter(s => s.todayStarId)` 신설 → Today 카드 목록·카운트 공통 소스로. `slice(0, 3)` 제거(오늘 별 있는 행성 **전부** 표시), 비었을 때 한 줄 안내(`.todayEmpty`) |
| [DashboardPage.module.css](../src/pages/DashboardPage.module.css) | `.todayEmpty` 스타일 추가 |
| [GoalStore.jsx](../src/store/GoalStore.jsx) | `loadBlackholeGalaxies()`(status='blackhole' 은하 조회)·`restoreGalaxy(id)`(active 복귀 + reload) 추가, context value/deps 반영 |
| [BlackHolePage.jsx](../src/pages/BlackHolePage.jsx) | 은하+행성 동시 로드(`Promise.all`), "보관된 목표"(은하)·"보관된 행성" 2섹션, 은하/행성 별도 선택 Set + 통합 되살리기, 빈 상태는 **둘 다 비었을 때만** |

## 4. 동작 (수정 후)

| 상황 | 전 | 후 |
|---|---|---|
| 보완으로 행성 추가 후 Today 카드 | 3개 초과분(새 행성) 잘림 | 오늘 별 있는 행성 **전부** 표시 → 새 행성 노출 |
| Today 카드 목록 vs 헤더 카운트 | 불일치 가능 | `todayStarId` 단일 소스로 일치 |
| 목표 포기 후 블랙홀 | 은하 안 보임·복구 불가 | "보관된 목표"에 노출 → 선택 후 되살리기로 복귀 |
| 은하 복구 | 불가 | 은하 active 전환, 소속 행성 동반 재등장 |
