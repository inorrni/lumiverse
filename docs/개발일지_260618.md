# 개발일지 — 2026.06.18

> [!note] 한 줄 요약
> Should 기능 2종을 붙였다 — **블랙홀 보관함**(보관된 행성 조회·복원)과 **별자리**(누적 별 14개↑ 보상: LLM 상징 추천 + 누적 별을 점·선으로 잇는 **절차적 SVG 그리기**). 더해 대시보드 **이전 목표 복원**·Today/Mid-Check 활성목표 매칭, **전역 커서** 정리. 브랜치 `feat/blackhole-restore`.

## 1. 대시보드 — 이전 보던 목표 복원 + 카드 활성목표 매칭 (feat)
- 캐러셀 활성 목표 id를 `localStorage`에 저장 → 다른 페이지에서 `/app` 복귀 시 그 목표로 복원(진입 1회, ref 가드).
- Today·Mid-Check 카드를 `featured`(goals[0]) → **현재 활성 목표** 기준으로 표시.

## 2. 전역 커서 정리 (style)
- `body { cursor:default }`로 텍스트의 I-beam 제거 → 화살표. `a/label/select/summary/[role=button]` 포인터, 비활성 버튼 화살표, `input/textarea` 텍스트 커서. 클릭 span(.viewAll) 포인터.

## 3. 블랙홀 보관함 (feat) — 행성 전용
- `/app/blackhole` → `BlackHolePage`(ComingSoon 대체). 보관된 행성(`status='blackhole'`, 부모 목표 active만) 목록 — 태그·소속 목표명·보관일.
- **선택(다중) → 하단 "되살리기"** → `restorePlanets`(status=active) → 우주맵/투두 재등장. 영구 삭제 없음(기획 "사라지지 않아요").
- 스토어: `loadBlackholePlanets`·`restorePlanets`.

## 4. 별자리 (feat) — 누적 별 14개↑ 보상
> [!important] 조건 정정
> 기존 `canConst = goal.stars >= 14`(배정된 총 별)는 기획과 어긋남 → **누적 체크 별 `starsEarned >= 14`**(기획 §5/§7)로 정정.
- **그리기**: `ConstellationArt` — 누적 별(`star_count`)을 황금비 quasi-random(목표 id 시드)으로 흩뿌려 **점+선 SVG**로 그림. 결정적(같은 목표=같은 모양), `currentColor`(먹/갱지 적응). UniverseMap 패턴 재사용.
- **상징**: `constellation-symbol` Edge Function(trigger=`symbol`) → 이모지 후보 4~5개 → `ConstellationModal`에서 선택 → `createConstellation`(은하당 1개, `star_count`=생성 시점 누적 별) → 완성 연출.
- 대시보드 "별자리 만들기"(earned≥14) → 모달 / 완성 시 카드·우주 배지에 별자리 그림+상징. 재생성 불가(unique).
- 마이그레이션 불필요(`lumiverse_constellations`·RLS 기존).

## 5. 검증 · 형상관리
- `npm run build` ✅ · `deno check`(constellation-symbol) ✅ · `functions deploy constellation-symbol` ✅ — OPTIONS 200 / POST 401.
- 커밋(타입별): `feat`(대시보드 복원) / `style`(커서) / `feat`(블랙홀) / `feat`(별자리). GoalStore 공유분은 revert/reapply로 블랙홀·별자리 커밋 분리.

> [!warning] 미검증 / 범위 밖
> 사람 손 QA(블랙홀 복원·별자리 모달/그리기/시드 결정성·먹갱지 대비) 별도 필요. 목표 단위 블랙홀 조회, 생성 애니메이션(Could), 행성상세 별자리 안내는 향후.

## 6. 은하계 클릭 → 목표 페이지 UX 개선 (feat) — `feat/universe-page-visual`

> [!note] 한 줄 요약
> 은하계를 누르면 열리는 목표 상세 페이지(행성 목록)의 시각·인터랙션을 다듬었다: **행성 이미지 조건부 표시**, **섹션 레이블 변경**, **행성 클릭 → 세부목표 페이지 이동**, **행성 아이콘 + 별 개수 표기**.

### 6-1. PlanetDetailPage — 목표 레벨(은하 클릭) 뷰 개선
- **행성 이미지**: step(세부목표) 레벨일 때만 표시. 목표 레벨에서는 제거(`{step && <Planet />}`).
- **섹션 레이블**: '오늘의 별' → **'행성'**. 해당 뷰가 세부목표(행성) 목록임을 명확히.
- **항목 클릭 네비게이션**: 각 행성 항목 클릭 → `/app/planet/${s.id}` (세부목표 상세). `onPress` prop 전달.
- **이모티콘 제거**: `sub={s.detail}` 제거 → 행성 이름만 표시.

### 6-2. CheckRow — `onPress` 분리 모드 추가
- `onPress` prop 추가. 제공 시 별 토글 버튼 제거, 좌측 **정적 행성 아이콘** + 본문 클릭은 네비게이션으로 분리.
- `onPress` 없을 때: 기존 `.main` 버튼(star 토글) 그대로 유지 → TodayPage 등 기존 사용처 영향 없음.
- 우측 count 앞에 ★ 아이콘 추가(행성 목록 전용) → "★ 3/7" 형태로 별 개수임을 명시.

### 6-3. `PlanetIcon` 추가 (`icons.jsx`)
- 채워진 원(행성 몸통) + 기울어진 타원 고리 SVG. `currentColor` → 테마 자동 적응.
- CheckRow `onPress` 모드 좌측 정적 아이콘 전용.

# 개발일지 (후속) — 2026.06.18 · 2차

> [!note] 한 줄 요약
> **가입 후 닉네임 온보딩**(트리거가 nickname을 비워 두고 → `/welcome`에서 직접 입력 → 프로필 저장)을 붙이고, 앱이 닉네임으로 부르게 했다(인사·self-insight). 대시보드 디테일도 다듬음 — 시간대별 인사, 헤더 알람/사용자 버튼, 별자리칸 문장화, Today "체크/전체·D-day", Mid-Check는 저장된 점검값 표시. 브랜치 `feat/nickname-onboarding`.

## 8. 닉네임 온보딩 (feat)
- 마이그레이션: `lumiverse_handle_new_user` 트리거가 nickname **자동 채움 중단**(null) → 신규만 미설정. 기존 유저 영향 없음.
- `AuthStore`: 세션 후 `profiles.nickname` 조회(`nickname`·`profileReady`)·`updateNickname`.
- `WelcomePage`(`/welcome`): 닉네임 입력(2~12자) → 저장 → `/mode`.
- `RequireAuth`: `profileReady && !nickname && path!=='/welcome'` → `/welcome` 가드(조회 전 보류로 깜빡임 방지).

## 9. 닉네임 호명 — self-insight (feat)
- 함수 프롬프트가 "당신" 대신 **"{닉네임}님"** 으로 호명(없으면 폴백), 캐시 키에 닉네임 포함. `api`·`UniverseCheckPage`가 닉네임 전달. 재호출 `sig`를 목표 내용 전체로 확장(한줄평 텍스트 변경도 갱신).

## 10. 대시보드 디테일 (feat)
- 인사: 시간대별(morning/afternoon/evening/night) + 닉네임. 헤더 알람(기본 틀)·사용자(→설정) 버튼화.
- 별자리칸: 목표명 제거, 상황 문장(완성/가능/N개 더). 설정의 별자리 토글은 **유지**(제거했다 복원).
- Today: 헤더 `오늘체크/전체 · D-day`, 줄은 제목만.
- Mid-Check: 저장된 최신 점검 메시지 표시(LLM 재호출 X — `midchecks` 임베드→`deriveGoal` 최신 1개), 버튼 "중간점검". 저장값 없으면 배지·메시지 모두 "점검 전"으로 일치.

## 11. 리뷰 반영 (fix)
- Mid-Check 배지(라이브 상태) ↔ 메시지("점검 전") 모순 → 저장값 기준 통일.
- self-insight 재호출 `sig`가 개수만 봐 한줄평 텍스트 수정 미반영 → 목표 내용 전체로 확장.
