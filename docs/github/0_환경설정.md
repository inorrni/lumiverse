# 0. 환경설정 A to Z

내 컴퓨터를 처음 세팅하는 단계입니다. **한 번만** 하면 됩니다. 위에서부터 순서대로 따라오세요. 막히면 맨 아래 "막혔을 때" 참고.

> [!tip] 큰 그림
> 우리는 **코드 = Claude Code로 작성**, **git = Claude Code에게 자연어로 지시**합니다.
> 그래서 git 명령어를 외울 필요는 없지만, 프로그램은 깔아둬야 합니다.

---

## 1단계 · 계정과 초대

1. **GitHub 계정** 만들기 → <https://github.com/signup>
2. 본인 GitHub 아이디(또는 이메일)를 **저장소 소유자에게 알려주기**.
3. 소유자가 협업자로 초대하면, **GitHub 알림/이메일에서 초대 수락**.
   - 수락해야 우리 저장소(`inorrni/lumiverse`)에 푸시할 수 있습니다.

---

## 2단계 · 프로그램 설치

| 프로그램 | 용도 | 받는 곳 |
|---|---|---|
| **Git** | 버전관리 엔진 (Claude가 내부에서 사용) | <https://git-scm.com/downloads> |
| **Node.js (LTS 버전)** | 앱 실행 (`npm`) | <https://nodejs.org> — "LTS" 쪽 |
| **VS Code** | 코드 에디터 | <https://code.visualstudio.com> |
| **Claude Code** | 우리의 주력 도구 | <https://claude.com/claude-code> |

설치 후 터미널(맥: 터미널 앱 / VS Code 내장 터미널)에서 아래로 확인:

```bash
git --version     # git version 2.x.x 처럼 나오면 OK
node --version    # v20.x 처럼 나오면 OK
```

**Git 최초 1회 설정** (커밋에 내 이름이 찍히도록):

```bash
git config --global user.name "본인이름"
git config --global user.email "본인GitHub이메일"
```

> [!tip] 직접 치기 싫으면
> Claude Code를 켠 뒤 **"git 사용자 이름/이메일을 OO / xx@메일 로 설정해줘"** 라고 해도 됩니다.

## 3단계 · 프로젝트 내려받기 (클론)

1. GitHub 저장소 페이지에서 초록색 **`Code` 버튼 → HTTPS 주소 복사**.
2. 프로젝트를 둘 폴더로 이동한 뒤:

```bash
git clone https://github.com/inorrni/lumiverse.git
cd lumiverse
```

> [!tip] Claude Code로 하기
> 빈 폴더에서 Claude Code를 켜고 **"이 저장소 클론해줘: https://github.com/inorrni/lumiverse.git"** 라고 해도 됩니다.
> 첫 푸시 때 GitHub 로그인 창이 뜨면 본인 계정으로 인증하세요.

---

## 4단계 · 실행해보기

> [!note] 아직 앱 코드가 없을 수 있어요
> 초기엔 `package.json`이 없을 수 있습니다. 그러면 이 단계는 코드가 올라온 뒤에 합니다.

`package.json`이 있다면:

```bash
npm install      # 의존성 설치 (node_modules 생성 — 커밋 안 됨)
npm run dev      # 로컬 개발 서버 실행 → 안내되는 http://localhost:5173 접속
```

## 5단계 · 환경변수 파일 만들기

API 키 같은 비밀값은 코드에 직접 쓰지 않고 `.env.local`에 둡니다. **이 파일은 절대 커밋되지 않습니다.**

```bash
cp .env.example .env.local
```

그다음 `.env.local`을 열어 본인 값(Supabase URL·anon key, 백엔드 API 주소)을 채웁니다.

> [!warning] 중요
> - `.env.example`(예시·값 없음)은 커밋 O / `.env.local`(실제 값)은 커밋 X.
> - 이 프론트엔드 `.env.local`에는 **공개돼도 안전한 값(Supabase anon key 등)만** 둡니다. **LLM 키(Groq·Solar)는 별도 백엔드 리포의 서버 환경변수**로만 보관합니다. 자세히는 → [2_보안_체크리스트.md](2_보안_체크리스트.md)
