## 1. 대시보드 — 이전 보던 목표 복원 + 카드 활성목표 매칭 (feat)
- 캐러셀 활성 목표 id를 `localStorage`에 저장 → 다른 페이지에서 `/app` 복귀 시 그 목표로 복원(진입 1회, ref 가드).
- Today·Mid-Check 카드를 `featured`(goals[0]) → **현재 활성 목표** 기준으로 표시.

## 2. 전역 커서 정리 (style)
- `body { cursor:default }`로 텍스트의 I-beam 제거 → 화살표. `a/label/select/summary/[role=button]` 포인터, 비활성 버튼 화살표, `input/textarea` 텍스트 커서. 클릭 span(.viewAll) 포인터.

## 3. 블랙홀 보관함 (feat) — 행성 전용
- `/app/blackhole` → `BlackHolePage`(ComingSoon 대체). 보관된 행성(`status='blackhole'`, 부모 목표 active만) 목록 — 태그·소속 목표명·보관일.
- **선택(다중) → 하단 "되살리기"** → `restorePlanets`(status=active) → 우주맵/투두 재등장. 영구 삭제 없음(기획 "사라지지 않아요").
- 스토어: `loadBlackholePlanets`·`restorePlanets`.

## 4. 별자리 (feat) — 누적 별 14개↑ 보상
> [!important] 조건 정정
> 기존 `canConst = goal.stars >= 14`(배정된 총 별)는 기획과 어긋남 → **누적 체크 별 `starsEarned >= 14`**(기획 §5/§7)로 정정.
- **그리기**: `ConstellationArt` — 누적 별(`star_count`)을 황금비 quasi-random(목표 id 시드)으로 흩뿌려 **점+선 SVG**로 그림. 결정적(같은 목표=같은 모양), `currentColor`(먹/갱지 적응). UniverseMap 패턴 재사용.
- **상징**: `constellation-symbol` Edge Function(trigger=`symbol`) → 이모지 후보 4~5개 → `ConstellationModal`에서 선택 → `createConstellation`(은하당 1개, `star_count`=생성 시점 누적 별) → 완성 연출.
- 대시보드 "별자리 만들기"(earned≥14) → 모달 / 완성 시 카드·우주 배지에 별자리 그림+상징. 재생성 불가(unique).
- 마이그레이션 불필요(`lumiverse_constellations`·RLS 기존).

# 개발일지 (후속) — 2026.06.18 · 3차

> [!note] 한 줄 요약
> ① 전 페이지 상단 뒤로가기를 **목적지 하드코딩 → 실제 이전 페이지(`navigate(-1)`) + 아이콘만**으로 통일. ② 우주 탭(UniversePage)을 2D CSS scatter에서 **Three.js(R3F) 실시간 인터랙티브 3D 우주**로 전면 교체 — 은하계=목표, 큰 점=행성(step), 작은 점=별(누적). 회전/줌/팬 + 탭 진입, 빈티지 일러스트·별모양·둥근 타원 배치까지 반복 튜닝.

## 13. 전역 뒤로가기 → 이전 페이지 통일 (refactor)
- `BackRow`는 이미 `to` 없으면 `navigate(-1)` 폴백 → 앱 내부 페이지들의 `to="/app"` 등 **하드코딩 목적지 제거**해 직전 화면으로 가게.
- 라벨 옵션화: `BackRow`의 기본 `label='뒤로'` 제거 → 라벨 있을 때만 `<span>` 렌더, 없으면 **화살표 아이콘만**(`aria-label`은 '뒤로 가기' 폴백).
- 적용: Today·Universe·Settings·PlanetDetail·MidCheck(결과)·BlackHole·UniverseCheck. **제외**: 온보딩 3종(Mode/GoalInput/Plan — 입력값 복원 `state` 유지), MidCheck 보충모드(같은 페이지 모드 토글), DashboardPage(뒤로 버튼 없음).

## 14. UniversePage → Three.js 3D 우주 (feat)
> [!important] 스택 추가
> `three@0.184` · `@react-three/fiber@8`(React 18 호환, v9는 React 19 필요) · `@react-three/drei@9`. 무거운 번들은 `React.lazy`로 **코드 스플릿**(우주 탭 진입 시에만 ~900KB 청크 로드).

### 14-1. 구조 (신규 파일)
- `lib/seededRandom.js` — `mulberry32`/`hashSeed`/`rngFor`. 목표 id 시드로 **결정적**(같은 목표=항상 같은 우주).
- `feature/universe/galaxyLayout.js` — `galaxyPositions`(은하 배치) · `planetLayout`(행성) · `starCloud`(별 구름) · `galaxyExtent`(콘텐츠 반경) · `splitByShape`(별 모양 분리).
- `feature/universe/dotTexture.js` — 런타임 canvas로 글로우 원형 + **별 모양 3종** 텍스처 생성·캐시(외부 에셋 무의존 → Pages base 영향 X).
- `feature/universe/Galaxy3D.jsx` — 단일 은하(행성 스프라이트 + 별 Points + 중심 글로우 + 자전/hover/클릭).
- `feature/universe/UniverseScene.jsx` — `<Canvas>`(투명) + 라이트 + `OrbitControls` + 텍스처 로드(Suspense).
- `pages/UniversePage.jsx`/`.module.css` — scene(flex로 채움) 위, footer(블랙홀 보관함) 하단 고정.

### 14-2. 데이터 매핑
- 은하계=`goal`, 큰 점=`step`(행성), 작은 점=별. 별은 배정 총량(`goal.stars`)을 흩뿌려 앞쪽 `starsEarned`개=**체크(밝은톤)** / 나머지=**미체크(중간톤)**.
- 행성 크기=진행도, 글로우/또렷함=`clarity`. 색은 `useTheme()`로 ink(아이보리 발광·additive) / paper(먹점) 분기.

### 14-3. 인터랙션
- `OrbitControls`: 회전 + 줌 + **팬(`enablePan`)**(처음엔 중심 고정이라 옆으로 못 옮긴다는 피드백 반영).
- **드래그 vs 탭 구분**: 빈 공간이 거의 없어 회전 드래그가 자꾸 행성 클릭으로 새던 문제 → `onPointerDown` 좌표 기록 후 `onClick`에서 이동량 **6px 이하**일 때만 `navigate('/app/planet/:id')`. 판정 구도 `extent*0.82`로 축소해 빈 공간 확보.
- **목표명 라벨**: 3D 따라다니는 `<Html>`은 줌 시 사라져 → **화면 상단 중앙 고정 DOM 오버레이**로 변경(hover한 은하 이름, 줌/회전 무관하게 또렷).

### 14-4. 비주얼 튜닝 (피드백 반복)
- **별 모양**: 부드러운 점 → **얇고 긴 스파클 3종**(4/5/6각, inner 0.06/0.10/0.12). 이모티콘처럼 통통하던 것 → 바늘형, 중심 글로우 축소.
- **행성 빈티지**: 3D 구 → `planet_2.png` **빌보드 스프라이트**(항상 카메라 향함). 스케일을 **이미지 실제 가로:세로 비율**(`tex.image` aspect)에 맞춰 **원 찌그러짐 해결**(square 고정이 원인이었음).
- **은하 배치**: 평면 디스크 → **세로 70% 둥근 타원체**(피보나치 구 분포).
- **행성 배치**: 인덱스마다 반지름 증가(삼각형처럼 규칙적) → 방향·거리 **완전 seed 랜덤**으로 자유 산개.
- **중심 글로우**: 콘텐츠 반경(`galaxyExtent`)에 맞춰 감싸되, 행성·별을 압도하지 않게 **불투명도 대폭 축소**(테두리 페이드는 유지).
