# 평가보고서 — 2026.06.17

> [!abstract] 종합
> 어제 구조적 안정화(인증 가드·FK 23503)를 마친 온보딩 흐름의 **UX 정합성 결함**을 정리한 세션. 입력방식에 따라 무의미하던 선택지를 감추고(조건부 노출), 동작과 어긋나던 라벨·라우트명을 바로잡고, 화면 전환 시 사용자 입력이 보존되도록 router state를 양방향화했다. **기능 추가보다 흐름 품질·일관성 개선**에 해당하며, 빌드·정적 검증 통과. 남은 관문은 동일하게 **사람 손 QA·Should 기능**.

---

## 1. 평가 기준

1. UX 정합성 — 사용자가 보는 선택지·라벨·전환이 실제 동작과 일치하는가
2. 상태 관리 — 화면 전환 중 입력값이 의도대로 보존/초기화되는가
3. 코드 품질 — 네이밍·하위호환·중복 제거가 적절한가

---

## 2. 항목별 평가

| 항목 | 평가 | 근거 | 점수 |
|---|---|---|---|
| **UX 정합성** | 우수 | 죽은 UI(강도) 조건부화, 거짓 라벨·라우트명 정정으로 화면=동작 일치 회복 | 9/10 |
| **상태 관리** | 우수 | 라우트 형제 간 `location.state` 양방향 계약 확립. 뒤로가기·"다시입력" 양쪽 보존 규칙 정의 | 9/10 |
| **코드 품질** | 우수 | `git mv`로 히스토리 보존, `BackRow.state` 선택적 prop으로 하위호환, 중복 SVG 제거 | 9/10 |
| **검증** | 양호 | 빌드·grep·하위호환 정적 확인. 사람 손 클릭 QA는 미수행 | 7/10 |

---

## 3. 변경의 효과

> [!success] 개선
> - **죽은 UI 제거**: "알아서" 모드에서 아무 영향 없던 강도 선택을 감춰 인지 부하·오해 제거
> - **화면 = 동작 일치**: "분해받기" 라벨/`breakdown` 라우트가 직접입력 흐름과 어긋나던 모순 해소
> - **입력 보존**: 온보딩 뒤로가기 시 목표·디데이 손실 차단 → 재입력 마찰 제거
> - **재사용 패턴**: `BackRow.state`는 다른 화면에서도 뒤로가기 시 컨텍스트 전달에 활용 가능

> [!warning] 한계
> - 사람 손 클릭 QA 미수행(빌드·정적 검증만) — 전진/후진 반복, "알아서"↔"추천" 토글 시나리오 실측 권장
> - 자동화 테스트 여전히 0 — 이런 상태 보존 회귀는 라우팅 통합 테스트로 잡기 좋은 후보
> - Should(중간점검·블랙홀·별자리) 미착수는 어제와 동일

# 평가보고서 (후속) — 2026.06.17 · 2차

> [!abstract] 종합
> PR #15 머지 후 온보딩 후속 작업 3건: **D-Day 필수화 · 입력 포커스 테두리 정리 · 리프레시(다른 분해 추천)**. 입력 검증·에러 위치·포커스 시각화 같은 **폼 UX 디테일**과, 캐시까지 고려한 **풀스택 기능(프론트+Edge Function)**을 더했다. 빌드 통과, 단 Edge Function은 **재배포 전 미적용**·사람 손 QA 미수행이 남은 관문.

## 6. 후속 항목별 평가

| 항목 | 평가 | 근거 | 점수 |
|---|---|---|---|
| **폼 검증(D-Day 필수)** | 우수 | 필수화 + 에러 위치 버그(목표 칸 오표시) 동시 해소. 다운스트림 폴백 영향 점검 | 9/10 |
| **포커스 시각화** | 우수 | 전역 링이 내부 input에만 걸리던 근본 원인 추적 → `.wrap` 일임으로 공용 컴포넌트 전반 개선 | 9/10 |
| **리프레시(풀스택)** | 양호 | 랜덤 대신 "제외" 방식으로 체감 차이 보장 + 캐시 키 정합. 단 **Edge Function 재배포 의존**·deno 로컬 검증 부재 | 8/10 |

> [!success] 강점
> - 폼 디테일(필수·에러 위치·포커스)을 **공용 컴포넌트 레벨**에서 고쳐 파급 효과 큼
> - 리프레시를 캐시 키까지 고려해 설계 — 단순 reroll의 "같은 결과" 함정 회피

> [!warning] 한계 / 리스크
> - **Edge Function 미배포 시 리프레시 무효**(캐시 히트로 동일 결과) — 배포가 선결
> - 로컬 `deno` 부재로 함수 타입체크 미수행(배포 시 검증)
> - 리프레시 연타 시 `RATE_LIMIT_PER_HOUR=30` 도달 가능
> - **기획 변경(디데이 자유→필수)** 문서 반영 미완
> - 사람 손 클릭 QA·자동화 테스트 여전히 미수행
