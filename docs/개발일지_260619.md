# 개발일지 — 2026.06.19

> [!note] 한 줄 요약
> 별 개수 모델을 **'디데이까지 남은 일수'(당일 제외) → '오늘부터 디데이까지'(당일 포함)** 로 정정. 데일리 체크 습관 모델에서 목표일 당일에도 체크할 별이 있어야 자연스럽고, "오늘까지" 같은 **당일 목표**도 만들 수 있게 됨. 브랜치 `feat/dday-inclusive-stars`.

## 1. 배경 — 왜 바꿨나

- 0618 9차에서 별 개수 off-by-one(생성이 미리보기보다 1개 많음)을 **미리보기(`daysUntil`)에 맞춰** 디데이 당일 제외(`[오늘…디데이 전날]`)로 정정했었다.
- 그 결과 시작(오늘)은 포함·끝(디데이)은 제외라 **비대칭**이 됐고, 두 가지 문제가 남았다:
  1. **목표일 당일에 체크할 별이 없음** — 데일리 체크 앱인데 마지막 날(목표일)에 할 일이 비어 "마지막 한 푸시"가 빠진 느낌.
  2. **당일 목표 생성 불가** — 디데이=오늘이면 `daysUntil`이 null → 생성 자체가 막힘("오늘까지 끝낼 일"을 등록 못 함).
- 결정: 디데이를 **목표일(작업하는 마지막 날)** 로 보고 별 생성에 포함. "오늘부터 디데이까지(양끝 포함)".

## 2. 변경 — 카운트다운과 별 개수의 의미 분리

> [!important] 핵심
> `daysUntil`은 화면의 **D-N 카운트다운**(D-1, D-3…)에도 쓰여서, 그대로 +1 하면 "내일이 디데이인데 D-2"처럼 카운트다운이 깨진다. **두 의미를 분리**했다.

- **`daysUntil(dday)`** ([date.js](../src/lib/date.js)) — D-N 카운트다운용. `Math.round`로 정수화, **당일=0(D-DAY)**·미래=N·과거/미설정=null (기존 `>0`→`>=0`, `ceil`→`round`).
- **`starCount(dday)`** (신설) — 별 개수용. `daysUntil + 1` = 오늘부터 디데이까지 양끝 포함. 당일 목표=별 1개, 과거/미설정=null.

## 4. 동작 (오늘 06-19 기준)

| 디데이 | 별 개수(전) | 별 개수(후) | D-표시 |
|---|---|---|---|
| 06-19 (당일) | 생성 불가 | **1개** | D-DAY |
| 06-20 (내일) | 1개 | **2개** | D-1 |
| 06-22 | 3개 | **4개** | D-3 |

별 생성 범위는 `enumerateDatesISO(start, end)`(양끝 포함)에 `end=디데이`를 그대로 넘겨 `starCount`와 항상 일치한다(미리보기=생성 개수 규칙 유지).
