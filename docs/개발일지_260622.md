# 개발일지 — 2026.06.22

> [!note] 한 줄 요약
> "버전1(모바일 PWA)" 확정 후 **웹버전 개발**에 착수. 클로드 디자인(`SNR Universe.dc.html`)을 참고해 **반응형 데스크톱 대시보드**(사이드바 + 1120px 그리드, 다크)를 1차 구현했으나, 결과물이 컨셉·완성도 기대와 맞지 않아 **전면 원복**했다. v1은 그대로 보존(리스크 0).

## 1. 시도한 것 (원복됨)
- **분기 전략**: `useMediaQuery('(min-width: 1024px)')` 훅 + `DashboardPage`를 얇은 래퍼로 — 데스크톱이면 `WebDashboard`, 아니면 기존 모바일(`MobileDashboard`). 브레이크포인트 넘나들 때 한쪽만 마운트(훅 규칙 안전).
- **WebDashboard**: 자체 셸(모바일 `AppScreen` 비의존, 후속 분리 대비)로 사이드바·헤더·Hero 우주 카드(진행률·궤도·코어글로우·행성·블랙홀·잠긴 별자리)·오늘의 별 캐러셀·우측 레일(AI 코치·성장 현황·빠른 메뉴)·하단 3열을 디자인대로 옮김.
- **실데이터 매핑**: `useGoals`로 진행률(평균 선명도)·행성(목표)·오늘의 별 토글·성장 현황·별자리 진행을 연동. **Pretendard**는 데스크톱 진입 시에만 동적 로드(모바일 번들 부담 0).

## 2. 왜 원복했나
> [!important] 판단
> 디자인을 충실히 옮겼으나 **방향성 자체가 제품 컨셉/완성도와 맞지 않는다**고 판단(사용자 피드백). 부분 수선보다 **전면 원복 후 방향 재정의**가 비용·리스크 면에서 합리적.

- 원복은 **표시 레이어 신규 추가분만 제거** → v1 모바일 앱에 영향 0.
- 무엇이 별로였는지(레이아웃 구조 / 다크 톤 / 행성 비주얼 / 정보 밀도 등) **구체화 후 재시도**가 다음 단계.
