## 3. 만든 것

### 3-1. 프로젝트 구조
```
src/
├── styles/         디자인 토큰(tokens.css) + 리셋(base.css)
├── components/
│   ├── ui/         atom·molecule 12종 (Button, TextInput, Card, DataView,
│   │               ClarityBar, Toggle, RadioRow, Kicker, Wordmark, BackRow,
│   │               Celestial, icons)
│   ├── layout/     organism 3종 (AppScreen, CosmicBackground, BottomNav)
│   └── feature/    기능 그룹 (universe/UniverseMap, galaxy/PlanetRow,
│                   today/CheckRow·TodayRow)
├── pages/          화면 조립만 10종 (Landing, Auth, Mode, GoalInput,
│                   AiBreakdown, Dashboard, Today, PlanetDetail, Settings,
│                   ComingSoon)
├── store/          GoalStore(목표·체크 루프) · AuthStore(mock 인증)
├── hooks/          useAsync(4상태) · useLocalStorage
├── lib/            api(mock AI 분해) · errors(친화 메시지) · date(KST 안전)
└── sample/         디자인 레퍼런스(목업 11화면 + 캔버스) — /design 경로
```
앱 코드 규모: **컴포넌트/페이지 33개 · CSS Module 26개 · 라우트 13개**.

### 3-2. 디자인 토큰 (규칙 ①)
`LUM` 팔레트(흑백 먹 + 갱지)를 **CSS 변수 6종 카테고리**로 추출:
Color · Typography · Spacing(4px 그리드) · Radius · Shadow · Z-index.
모든 컴포넌트 CSS는 `var(--token)` 만 참조(raw hex/px 금지). 다크가 기본 테마.

### 3-3. 컴포넌트 분리 (규칙 ②)
`ui → layout → feature → pages` 계층. 페이지는 로직 없이 **조립만** 담당.
모든 인터랙티브 요소 터치 타깃 44px↑.

### 3-4. 4상태 UI (규칙 ③)
`DataView`(로딩·에러·빈·성공) + `useAsync` + `friendlyError`(401/429/500/네트워크 매핑).
AI 목표 분해 화면에서 4상태를 모두 구현. 로딩=스켈레톤, 에러=재시도, 빈=CTA.

### 3-5. 핵심 루프 (Must)
- `GoalStore.toggleStarToday`: 오늘의 별 체크 토글 → `done`/`lastCheck` 갱신 → **clarity 재계산**
- 대시보드 Daily Stars·행성 pip·행성 상세 선명도에 **즉시 반영**
- 같은 날 재탭 = 해제(롤백), 날짜 바뀌면 다시 체크 가능

### 3-6. 온보딩 플로우
랜딩(로그인 여부 분기) → 로그인(mock) → 모드(강도·입력방식) → 목표+D-Day → AI분해.
**알아서 모드**는 AI 호출을 건너뛰고 직접 행성 입력으로 분기.

## 4. 핵심 의사결정

1. **목업 보존** — `screens/`까지 `src/sample/`로 옮겨 디자인 레퍼런스로 남기고 `/design` 경로에서 열람. 실제 앱은 새로 구성.
2. **데이터 모델 2단계** — 목업의 3단계(은하→행성→별-task)를 MVP에서 **목표(은하)→세부목표(별)** 2단계로 단순화. 체크 단위 = 세부목표. 행성 상세 = 목표 상세로 매핑.
3. **CSS Modules + 토큰** — 매우 커스텀한 우주 미감(그레인·먹·갱지) 때문에 Tailwind 대신 CSS 변수 토큰 + 컴포넌트 스코프 CSS 채택.
4. **mock 우선** — AI 분해·인증은 인터페이스를 고정한 mock으로 구현(2-1에서 Solar/Claude·Supabase로 교체 예정). localStorage 영속으로 새로고침에도 우주 유지.
5. **개발 주석 제외** — 목업의 `Annot`(API 엔드포인트·폴백 등 개발 메모)은 사용자 UI에서 제거, 의미 있는 동작만 구현.

## 7. 트러블슈팅 메모

- **JSX는 `//` 주석이 안 통한다** — `// <DesignCanvasApp />`가 텍스트+엘리먼트로 파싱돼 컴포넌트가 그대로 마운트됐다. 진짜 제거하려면 라인을 지워야 함.
- **KST 타임존 버그** — `toISOString()`은 UTC라 한국시간 자정에 날짜가 하루 밀린다. `lib/date`에서 로컬 달력 기준으로 통일.
- **dynamic+static 동시 import 경고** — DesignCanvasApp을 main에서 정적 import 하면 코드 분리가 풀린다. `/design` 지연 로드만 남기니 별도 청크(61.8kB)로 정상 분리.

## 8. 형상관리 · 배포 (GitHub Pages)

### 8-1. 형상관리
- `main`에서 새 브랜치 `feat/mvp-core`를 따고 **용도별 4커밋**(chore 부트스트랩 / chore 목업 레퍼런스 / feat MVP / docs)으로 분류 → **PR #3 머지**로 main 반영.

### 8-2. "사이트에 안 보임" 원인
- GitHub Pages가 켜져 있었지만 **main 루트를 Jekyll(legacy)로 서빙** → Vite 소스(`index.html`+`/src`)는 빌드되지 않아 앱이 렌더되지 않음. **빌드 산출물을 배포**해야 함.
- 프로젝트 Pages는 하위경로(`/lumiverse/`)라 절대경로 자산이 깨짐.

### 8-3. 조치
- **Vite `base`** — 빌드 시에만 `'/lumiverse/'` (로컬 dev 는 `'/'`)
- **라우터 `basename`** — `import.meta.env.BASE_URL` 로 하위경로 라우팅
- **`asset()` 헬퍼** — `/assets/...` 절대경로를 `BASE_URL` 기준으로 보정(천체·성운 이미지)
- **GitHub Actions 배포 워크플로우** — main 푸시 시 `npm ci → build → Pages 배포`, SPA 새로고침 대비 `404.html`(= index 복사) 생성
- Pages 소스를 **legacy → GitHub Actions** 로 전환

### 8-4. 검증
- `npm run build` ✅ — `dist/index.html` 자산이 `/lumiverse/assets/...` 로 출력 확인
- 라이브 URL: **https://inorrni.github.io/lumiverse/** (머지 후 Actions 배포 완료 시 반영)

> [!note] 메모
> `BrowserRouter` 사용이라 딥링크 새로고침이 404가 될 수 있어 `404.html` 폴백으로 앱 셸을 다시 받게 처리. 추후 HashRouter 또는 서버 리라이트로 대체 가능.
