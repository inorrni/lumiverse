# 트러블슈팅 — 대시보드·메인페이지 (2026.06.21)

> [!note] 개요
> 메인페이지 개편 중 마주친 3건. D-1은 같은 라우트 인스턴스 재사용에서 온 클릭 데드, D-2는 분모 집계 오류, D-3은 이미지 로딩(미해결·환경 특정 필요). 공통 교훈: **"클릭이 안 된다"를 click-area/state/route/api로 분해**하면 원인이 빠르게 좁혀진다.

## D-1 · 행성상세에서 행성 재클릭 시 무반응

> [!bug] 증상
> 행성상세 목록에서 행성을 누르면 그 내용으로 잘 들어가지만, **뒤로 간 뒤 다시 누르면 클릭이 먹히지 않는다**.

**재현**
1. 우주(은하) → `/app/planet/은하id`(행성 목록) 진입.
2. 목록의 행성 클릭 → `/app/planet/행성id`(여정) 진입(정상).
3. 뒤로(`navigate(-1)`) → 행성 목록 복귀.
4. 같은/다른 행성 클릭 → **반응 없음**.

**진단 (4갈래 배제)**
- **API ✕** — `goals`는 앱 최상단 `GoalProvider`에 1회 로드, 이동 시 재호출·초기화 없음.
- **route ✕** — `navigate('/app/planet/…')`·`navigate(-1)`·라우트 매칭 정상.
- **click-area ✕** — 목록 위를 덮는 오버레이 없음(여정뷰 큰 Planet은 `pointer-events:none`, 은하뷰엔 없음).
- **state ○** — 행성·은하 상세가 **같은 라우트 `/app/planet/:id`** 를 공유. 행성↔여정 이동이 param만 바꾸므로 React Router가 `PlanetDetailPage` **인스턴스를 언마운트 없이 재사용**, `{step ? 여정 : 목록}` 분기를 같은 DOM 위에서 재조정. 이 in-place 재사용이 왕복 후 목록 클릭 타깃을 어긋나게 함.

**원인**
- 동적 상세 라우트인데 `:id`가 바뀌어도 **새로 마운트되지 않음**(키 부재). 같은 인스턴스에서 조건 분기 DOM만 토글되며 재조정.

**해결** — `:id`마다 강제 리마운트(동적 상세 라우트 표준 해법).

```diff
// PlanetDetailPage.jsx
- export default function PlanetDetailPage() {
-   const { id } = useParams()
-   ...전체 로직...
- }
+ export default function PlanetDetailPage() {
+   const { id } = useParams()
+   return <PlanetDetail key={id} id={id} />   // id마다 새 마운트
+ }
+ function PlanetDetail({ id }) { ...전체 로직(useParams→id prop)... }
```

**회귀 방지** — id가 바뀔 때마다 컴포넌트가 새로 마운트되어 화면·클릭 핸들러가 항상 새로 생성. `key`는 React의 표준 리마운트 신호라 부수효과 없음(배경은 seed 고정이라 시각 동일).

> [!warning] 미실증
> 헤드리스 환경에서 클릭 재현 불가 → **코드 진단 기반 수정**. 실기기에서 (행성→뒤로→재클릭) 확인 필요. 그래도 안 되면 핸들러 자체 호출 여부(콘솔 로그)로 추가 좁히기.

## D-2 · Daily Stars가 100%에 도달하지 못함

> [!bug] 증상
> 메인 최상단 **Daily Stars**의 `X / Y`에서 오늘 별을 다 체크해도 Y에 못 미친다.

**원인**
- 분모 `totalPlanets = allSteps.length`가 **모든 목표의 모든 행성**을 셈 → 오늘 별이 없는 행성(특히 **기간종료 목표**의 행성)까지 분모에 포함 → 분자(오늘 체크)가 결코 Y에 도달 못 함.

**해결**
```diff
- const totalPlanets = allSteps.length
+ const totalPlanets = allSteps.filter((s) => s.todayStarId).length // 오늘 별 있는 행성만
```
대시보드 캐러셀도 "오늘 별 있는 목표만"으로 좁혀, **노출 대상과 분모가 한 기준**에서 나오도록 통일.

**회귀 방지** — 분자(`checkedToday`)와 분모(`todayStarId`)가 같은 "오늘 별" 축을 공유 → 100% 도달 가능, 기간종료/완료 목표는 분모에서 자연 제외.

## D-3 · 이미지가 전부 깨짐 (미해결 · 환경 특정 필요)

> [!bug] 증상
> 어떤 환경에서 화면의 이미지가 전부 안 뜬다("이미지를 못 불러옴").

**조사 (진행 중)**
- 앱 이미지는 `asset(path) = import.meta.env.BASE_URL + path`로 base를 반영 — dev는 `/`, **build는 `/lumiverse/`**([vite.config.js](../vite.config.js)).
- 가장 흔한 원인: **빌드본을 루트에서 열기**. `/lumiverse/` 접두 자산이 루트 서빙 시 전부 404(앱 로직은 동작해 "클릭은 되는데 이미지만 깨짐"으로 보임).
- raw 경로(`asset()` 미경유)·CSS `url(assets/…)` 누락은 1차 grep에서 미발견(추가 확인 필요).

> [!tip] 공통 교훈
> - **클릭 데드는 click-area / state / route / api 4갈래로 분해**해 하나씩 배제하면 빠르다. 이번엔 무상태·2D 경로의 정상성을 보이고 "같은 라우트 인스턴스 재사용"으로 좁혀졌다.
> - **동적 상세 라우트(`/x/:id`)에서 항목↔항목 이동이 이상**하면 `key={id}` 리마운트를 먼저 의심.
> - **카운트는 분자·분모가 같은 축**에서 나오게(오늘 별) — 집계 불일치를 구조적으로 차단.
> - **"이미지만 깨짐 + 앱은 동작" = base 경로** 신호. 보는 환경부터 특정.
