# 품질점검 — 2026.06.18

> [!abstract] 요약
> "되긴 되는데 실서비스급인가?" 점검. **정적으로 측정 가능한 지표(번들·에셋 용량·npm audit·접근성/SEO·리렌더 위생·테스트)는 실측**했고, 가장 확실한 결함이던 **천체 PNG 과대 용량을 `sharp`로 최적화(−92~97%)** 했다. 브라우저 실측(S1 Network/Performance·S2 Profiler·S3 Lighthouse 점수)은 사람이 직접 떠야 하는 항목으로, 아래에 측정 위치·기준을 남긴다.

## S1. Chrome DevTools — Network / Performance

**정적 대체(실측)**
- 빌드 청크: `index` 479KB(gzip 143KB) · `UniverseScene`(three.js) **893KB(gzip 242KB)** — `React.lazy`로 **코드 스플릿**돼 비-3D 화면엔 0 부담.
- 직렬 호출: `MidCheckPage.jsx:133` 보완 시 `for…await blackholePlanet`(N 작아 경미, 다건이면 `Promise.all` 여지).
- LLM Edge Function(goal-breakdown·mid-check·self-insight·constellation-symbol)은 동일 입력 **캐시(input_hash)** + rate limit로 재호출 차단.

**사람 측정 TODO**
- Network `Fetch/XHR` → `Time` 정렬 → 1초+ 호출 식별(특히 첫 LLM 호출 TTFB).
- `/app/universe` 진입을 **Slow 3G**로 — three 청크 로딩 중 스켈레톤/폴백("우주를 그리는 중…") 노출 확인.
- Performance 녹화 — 3D 회전/줌 시 Long Task(50ms+) 여부(Bottom-Up).

## S3. Lighthouse — 4지표

**정적 대체(실측)**
- **Performance**: 이미지 −95% 최적화 완료(위 표). 3D 청크 lazy. CLS — `.scene` 고정 flex 높이, `<img>` width/height·sprite aspect 보정으로 레이아웃 점프 억제.
- **Accessibility**: ✅ `lang=ko`·`viewport`. 입력은 `TextInput`로 라벨 중앙화. 🟡 `CosmicBackground` `<img>` **alt 누락**(장식 → `alt=""` 권장).
- **Best Practices**: 🟡 `npm audit` — moderate(esbuild **dev 서버 전용**, 프로덕션 영향 미미) + high 1건. `console.*` 3건(전부 warn/error 정당).
- **SEO**: ✅ `title`·`description`. 🟡 **OG/canonical 없음** → 공유 카드·SEO 만점 미달.

**사람 측정 TODO**

## S4. AI 코드리뷰 + 5단계 디버깅 (이번 세션 적용 사례)

- **mid-check 일반 메시지 버그**: ①재현 "잘 놀기" 목표에 일반 조언 ②가설 프롬프트 맥락 부족 ③검증 폴백 아닌 LLM 실출력 확인 ④수정 목표성격 기반 프롬프트 ⑤회귀 `PROMPT_VERSION`으로 캐시 무효화 + 재배포. → 이슈 L5.
- **행성 스프라이트 찌그러짐**: 원인 = 스프라이트 scale `[s,s,1]` 고정(이미지 비율 무시) → `tex.image` aspect 반영으로 수정.
- **별자리 두세줄**: 원인 = 난수 점을 인덱스 순서로 연결(윤곽 정렬 부재) → 무게중심 각도순 닫힌 윤곽. → 이슈 L6.
