# 추후개발 — 대시보드·메인페이지 (2026.06.21)

> [!note] 개요
> 0621 메인페이지 개편에서 **이번 범위 밖으로 남긴** 후속 과제와 결정 보류 항목. 우선순위: 🔴 높음 / 🟡 보통 / ⚪ 낮음.

---

## 1. ✅ 이미지 깨짐 — 종료(코드 무관)
- **결론**: localhost 연결 끊김이 원인. 자산 경로/`asset()`/base 모두 정상, 코드 수정 없음. (O10 무효)
- cf. [[트러블슈팅_대시보드_260621]] D-3.

## 2. 🔴 진행 중 목표가 많을 때 대시보드 노출 정책
- **배경**: 이번엔 "오늘 별 있는 목표만"으로 기간종료/완료는 정리했지만, **진행 중 목표 자체가 수십 개**면 캐러셀이 여전히 길다.
- **후보**: ① 핀 고정(사용자가 대시보드에 띄울 목표 선택, DB `pinned` 필요) · ② 대표 N개 + 요약(진행 N개·오늘 별 X/Y) · ③ 임박순 정렬 + 더 보기.
- **결정 필요**: 핀 방식은 스키마 추가 동반 → 기획 합의 후 착수.

## 3. ✅ 행성상세 `key` 리마운트 — 실기기 정상 확인
- 코드 진단 기반 수정([[트러블슈팅_대시보드_260621]] D-1)을 0621 실기기에서 (행성→뒤로→재클릭) 정상 확인. 종료.

## 4. 🟡 우주탭 필터 확장
- **블랙홀 칩**: 현재 블랙홀은 별도 보관함 버튼 유지(별도 로드 경로). 필터 칩에 합치려면 `loadBlackholeGalaxies` 연동 필요.
- **완료(completed) 정의 정교화**: 지금은 `clarity===100`. 중간점검 `finish`(마무리 권유)와의 관계·표기 정리 후보.
- **다중 선택/조합**: 현재 단일 선택. 라이프사이클 × 중간점검 단계 조합 필터는 후속.

## 5. 🟡 모드/스타일 라벨 통일
- 카드 태그는 `추천 · 살살`/`스파르타`/`알아서`로 표기. 앱 다른 화면 용어(살살모드/스파르타모드, 알아서/추천)와 **표기 일관성** 최종 점검.
