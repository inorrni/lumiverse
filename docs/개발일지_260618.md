# 개발일지 — 2026.06.18

> [!note] 한 줄 요약
> Should 기능 2종을 붙였다 — **블랙홀 보관함**(보관된 행성 조회·복원)과 **별자리**(누적 별 14개↑ 보상: LLM 상징 추천 + 누적 별을 점·선으로 잇는 **절차적 SVG 그리기**). 더해 대시보드 **이전 목표 복원**·Today/Mid-Check 활성목표 매칭, **전역 커서** 정리. 브랜치 `feat/blackhole-restore`.

## 1. 대시보드 — 이전 보던 목표 복원 + 카드 활성목표 매칭 (feat)
- 캐러셀 활성 목표 id를 `localStorage`에 저장 → 다른 페이지에서 `/app` 복귀 시 그 목표로 복원(진입 1회, ref 가드).
- Today·Mid-Check 카드를 `featured`(goals[0]) → **현재 활성 목표** 기준으로 표시.

## 2. 전역 커서 정리 (style)
- `body { cursor:default }`로 텍스트의 I-beam 제거 → 화살표. `a/label/select/summary/[role=button]` 포인터, 비활성 버튼 화살표, `input/textarea` 텍스트 커서. 클릭 span(.viewAll) 포인터.

## 3. 블랙홀 보관함 (feat) — 행성 전용
- `/app/blackhole` → `BlackHolePage`(ComingSoon 대체). 보관된 행성(`status='blackhole'`, 부모 목표 active만) 목록 — 태그·소속 목표명·보관일.
- **선택(다중) → 하단 "되살리기"** → `restorePlanets`(status=active) → 우주맵/투두 재등장. 영구 삭제 없음(기획 "사라지지 않아요").
- 스토어: `loadBlackholePlanets`·`restorePlanets`.

## 4. 별자리 (feat) — 누적 별 14개↑ 보상
> [!important] 조건 정정
> 기존 `canConst = goal.stars >= 14`(배정된 총 별)는 기획과 어긋남 → **누적 체크 별 `starsEarned >= 14`**(기획 §5/§7)로 정정.
- **그리기**: `ConstellationArt` — 누적 별(`star_count`)을 황금비 quasi-random(목표 id 시드)으로 흩뿌려 **점+선 SVG**로 그림. 결정적(같은 목표=같은 모양), `currentColor`(먹/갱지 적응). UniverseMap 패턴 재사용.
- **상징**: `constellation-symbol` Edge Function(trigger=`symbol`) → 이모지 후보 4~5개 → `ConstellationModal`에서 선택 → `createConstellation`(은하당 1개, `star_count`=생성 시점 누적 별) → 완성 연출.
- 대시보드 "별자리 만들기"(earned≥14) → 모달 / 완성 시 카드·우주 배지에 별자리 그림+상징. 재생성 불가(unique).
- 마이그레이션 불필요(`lumiverse_constellations`·RLS 기존).

## 5. 검증 · 형상관리
- `npm run build` ✅ · `deno check`(constellation-symbol) ✅ · `functions deploy constellation-symbol` ✅ — OPTIONS 200 / POST 401.
- 커밋(타입별): `feat`(대시보드 복원) / `style`(커서) / `feat`(블랙홀) / `feat`(별자리). GoalStore 공유분은 revert/reapply로 블랙홀·별자리 커밋 분리.
