# 개발일지 — 2026.06.22

> [!note] 한 줄 요약
> "버전1(모바일 PWA)" 확정 후 **웹버전 개발**에 착수. 클로드 디자인(`SNR Universe.dc.html`)을 참고해 **반응형 데스크톱 대시보드**(사이드바 + 1120px 그리드, 다크)를 1차 구현했으나, 결과물이 컨셉·완성도 기대와 맞지 않아 **전면 원복**했다. v1은 그대로 보존(리스크 0).

## 1. 시도한 것 (원복됨)
- **분기 전략**: `useMediaQuery('(min-width: 1024px)')` 훅 + `DashboardPage`를 얇은 래퍼로 — 데스크톱이면 `WebDashboard`, 아니면 기존 모바일(`MobileDashboard`). 브레이크포인트 넘나들 때 한쪽만 마운트(훅 규칙 안전).
- **WebDashboard**: 자체 셸(모바일 `AppScreen` 비의존, 후속 분리 대비)로 사이드바·헤더·Hero 우주 카드(진행률·궤도·코어글로우·행성·블랙홀·잠긴 별자리)·오늘의 별 캐러셀·우측 레일(AI 코치·성장 현황·빠른 메뉴)·하단 3열을 디자인대로 옮김.
- **실데이터 매핑**: `useGoals`로 진행률(평균 선명도)·행성(목표)·오늘의 별 토글·성장 현황·별자리 진행을 연동. **Pretendard**는 데스크톱 진입 시에만 동적 로드(모바일 번들 부담 0).

## 2. 왜 원복했나
> [!important] 판단
> 디자인을 충실히 옮겼으나 **방향성 자체가 제품 컨셉/완성도와 맞지 않는다**고 판단(사용자 피드백). 부분 수선보다 **전면 원복 후 방향 재정의**가 비용·리스크 면에서 합리적.

- 원복은 **표시 레이어 신규 추가분만 제거** → v1 모바일 앱에 영향 0.
- 무엇이 별로였는지(레이아웃 구조 / 다크 톤 / 행성 비주얼 / 정보 밀도 등) **구체화 후 재시도**가 다음 단계.

## 5. 후속 — 별자리 상징물 추천 프롬프트 수정 (`constellation-symbol`)

> [!note] 한 줄 요약
> 별자리 상징물 추천이 목표와 무관한 **보편적 "○○의 별"** 라벨을 내놓던 문제(예: "포트폴리오 사이트 완성" → "빛나는 별·비상의 별·정밀한 별·연결의 별·견고한 별")를 프롬프트 수정으로 바로잡고, **추천을 5개→3개**로 줄였다.

### 5-1. 원인
[constellation-symbol/index.ts](../supabase/functions/constellation-symbol/index.ts) 시스템 프롬프트가 "4-5 candidates + emblem **vibe** 라벨(예: 꾸준한 불꽃)"만 요구 → LLM이 목표 도메인과 무관한 별/우주 톤의 일반 라벨을 생성. 캐시(`input_hash`)에 프롬프트 버전이 없어 재배포해도 같은 목표는 옛 결과 반환.

### 5-2. 변경
| 항목 | 전 | 후 |
|---|---|---|
| 개수 | 4-5개 | **정확히 3개** (`validate` `slice(0,3)`도 보장) |
| 라벨 | emblem vibe(보편) | **그 목표의 구체적 주제/도메인**에 밀착. "○○의 별/빛나는 별/꾸준한 불꽃" 류 보편 필러 **금지** |
| 이모지 | 제약 없음 | **목표의 실제 내용**을 연상시키는 이모지. 일반 ⭐🌟✨🌠🌌🪐(별·우주) **회피** |
| 예시 | 없음 | 서로 다른 도메인 2개(포트폴리오·마라톤) few-shot — "항상 실제 목표에서 도출, 예시로 기본값 삼지 말 것" 명시 |
| 캐시 | `goal|intensity` | **`PROMPT_VERSION='v2'` 포함** → 옛 캐시 자동 무효화(mid-check 패턴과 동일) |

### 5-3. 비고 · 검증
- **Edge Function**이라 로컬 vite 빌드로 안 잡힘 → **재배포 필요**: `supabase functions deploy constellation-symbol`. **0622 배포 완료.**
- 규칙은 **보편적**(모든 목표 적용). 포트폴리오·마라톤은 형식/톤을 보여주는 illustrative 예시일 뿐 도메인 고정 아님(2개 도메인으로 anchoring 방지).
