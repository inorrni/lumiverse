## A1. 천체 이미지 과대 용량 (상세)

**현재**
| 파일 | 용량 | 표시 크기 |
|---|---|---|
| `public/assets/bg/nebula.png` | 3.15MB | 배경 전면 |
| `public/assets/celestial/blackhole.png` | 2.64MB | 최대 34px |
| `public/assets/celestial/constellation.png` | 2.19MB | 66×32px |
| `public/assets/celestial/planet.png` | 1.57MB | 최대 92px |
| **합계** | **~9.5MB** | — |

**목표**: 합 300KB 이하. 파일 경로·명·.png 확장자·알파 채널 유지(코드 무수정). 표시 크기의 2배(레티나)로 리사이즈 + 압축(`sharp`/pngquant).

**사용 위치**: `src/components/ui/Celestial.jsx`(planet/blackhole/constellation), `src/components/layout/CosmicBackground.jsx`(nebula).

**경위**: PR #5(천체 이미지 교체, 머지 d116632) 이후 용량 급증 확인 → 팀원에게 최적화 PR 요청. 기획 아키텍처의 "경량 + Canvas 렌더" 지향과도 어긋나 우선 개선 대상.
