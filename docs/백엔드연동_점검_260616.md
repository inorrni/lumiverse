# 백엔드 연동 점검 — 2026.06.16

> [!abstract] 결론
> Supabase 연동 표준 프롬프트(CRUD/Realtime/Storage/폼/환경분리)와 대조한 결과,
> **적용 가능한 항목은 대체로 준수**, 성격상 불필요한 항목(Realtime·Storage)은 **의도적으로 생략**.

## 2. #1 조치 내역 (배포 env 주입)

문제: Vite는 빌드 시 `VITE_*` 를 **번들에 박는다**. repo Secrets 를 **등록만** 하면 빌드에 안 들어가고, 워크플로우 build step의 `env:` 로 **주입**해야 한다.

- **Secrets 등록** (Settings → Secrets and variables → Actions): `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`, `VITE_SITE_URL` ✅
- **워크플로우 주입** (`.github/workflows/deploy.yml`):
  ```yaml
  - run: npm run build
    env:
      VITE_SUPABASE_URL: ${{ secrets.VITE_SUPABASE_URL }}
      VITE_SUPABASE_ANON_KEY: ${{ secrets.VITE_SUPABASE_ANON_KEY }}
  ```

> [!note] 메모
> `VITE_SITE_URL` 시크릿은 등록돼 있으나 현재 코드는 참조하지 않음(카카오 redirect는 `window.location.origin + BASE_URL` 사용). 추후 redirect를 고정 URL로 쓰려면 코드에서 `import.meta.env.VITE_SITE_URL` 로 연결.

**검증 방법**: main 머지 → Actions 빌드 → 라이브(https://inorrni.github.io/lumiverse/)에서 로그인·분해 동작 확인. (브라우저 DevTools Network 에서 `<ref>.supabase.co` 호출이 보이면 주입 성공)
