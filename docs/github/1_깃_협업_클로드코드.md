# 1. Claude Code로 git 하기 (매일 쓰는 작업법)

git 명령어를 외울 필요 없습니다. **하고 싶은 걸 자연어로 말하면** Claude가 git을 대신 다룹니다. 이 문서의 프롬프트를 그대로 복사해 쓰세요.

> [!tip] 황금 규칙
> 1. 작업은 **항상 새 브랜치에서** (절대 `main`에서 바로 X)
> 2. **시작 전 `main` 최신화**
> 3. 모르겠으면 **멈추고 Claude에게 물어보기**

---

## 하루 작업 한 사이클

### ① 작업 시작 — 최신화 + 새 브랜치

```
main 최신화하고, feat/goal-input 이라는 새 브랜치 만들어줘.
```

- `feat/goal-input` 자리에 이번에 할 일에 맞는 이름을 넣으세요 (규칙은 [CONTRIBUTING.md](CONTRIBUTING.md) 2번).
- Claude가 `main`을 받아오고(pull) 새 브랜치를 만들어 그 위로 옮겨줍니다.

### ② 작업 — 코드/문서 수정

평소처럼 Claude에게 기능을 요청해 코드를 만듭니다. 중간중간 저장(커밋)하는 게 좋습니다.

### ③ 커밋 — 변경분 저장

CLAUDE.md / CONTRIBUTING.md와 동일한 규칙입니다. 이 문구를 쓰세요:

```
변경분을 용도별로 분류해서 커밋해줘.
한 커밋에 한 가지 목적만 — 타입(feat/fix/docs/...)이 다르면 커밋도 나눠.
메시지는 변경 내용 보고 Conventional Commits로 작성해줘.
```

### ④ 푸시 — 내 브랜치를 GitHub에 올리기

```
지금 브랜치를 origin에 푸시해줘.
```

### ⑤ PR 만들기 — "합쳐주세요" 요청

```
현재 브랜치로 PR 올려줘. 제목·본문은 변경 내용 보고 작성해줘.
```

- Claude가 PR 링크를 줍니다. GitHub에서 [PR 템플릿](../.github/pull_request_template.md)이 자동으로 채워진 걸 확인하세요.
- 이후 **상대가 리뷰 → 머지**합니다. (CONTRIBUTING.md 4번)

### ⑥ 다음 작업 — 다시 ①부터

머지가 끝나면 다시 "main 최신화하고 새 브랜치 만들어줘"로 시작합니다.

## 자주 쓰는 추가 프롬프트

| 하고 싶은 것 | 이렇게 말하기 |
|---|---|
| 내 변경사항 현황 보기 | "지금 git 상태 보여줘 (뭐가 바뀌었는지)" |
| 남이 올린 main 변경 받기 | "원격 main 최신 변경을 가져와서 내 브랜치에 반영해줘" |
| 마지막 커밋 메시지 고치기 | "방금 한 커밋 메시지를 OO로 바꿔줘 (아직 푸시 전이야)" |
| 실수로 추가한 파일 빼기 | "이번 커밋에서 OO 파일은 빼줘" |
| 작업 잠시 치워두기 | "지금 작업 임시로 저장(stash)해두고 main으로 가줘" |

## ⚠️ 안전 수칙 (꼭 읽기)

git에는 **되돌리기 어려운 위험한 명령**이 있습니다. Claude에게 미리 이렇게 부탁해두면 안전합니다:

```
앞으로 git 작업할 때, 아래는 실행 전에 반드시 나한테 먼저 설명하고 확인받아줘:
- 강제 푸시 (push --force)
- main 브랜치에 직접 푸시
- reset --hard / 커밋이나 변경분을 삭제하는 작업
- 브랜치 삭제
그리고 위험하다고 판단되면 그냥 멈추고 물어봐.
```

- **`main`에 직접 푸시는 막혀 있습니다** (관리자가 보호 설정함) — 그래서 큰 사고는 구조적으로 차단됩니다.
- 그래도 잘 모르는 명령이 나오면 **"이거 안전해? 무슨 뜻이야?"** 라고 먼저 물어보세요.

## git 용어 미니사전

| 용어 | 한 줄 설명 |
|---|---|
| **repository (repo)** | 프로젝트 저장소. GitHub에 있는 우리 `lumiverse`. |
| **clone** | 저장소를 내 컴퓨터로 통째로 복사해 가져오기. |
| **commit** | 변경사항을 "이 시점"으로 저장 (사진 한 장 찍기). |
| **branch** | 작업용 평행 세계. `main`을 건드리지 않고 따로 작업. |
| **main** | 항상 동작하는 기준 본선. 직접 수정 X. |
| **push** | 내 커밋을 GitHub(origin)에 올리기. |
| **pull** | GitHub의 최신 변경을 내 컴퓨터로 받기. |
| **origin** | 우리 GitHub 원격 저장소를 가리키는 별명. |
| **PR (Pull Request)** | "내 브랜치를 main에 합쳐줘" 요청 + 리뷰 공간. |
| **merge** | 브랜치를 main에 합치기. |
| **conflict** | 같은 부분을 둘이 다르게 고쳐 git이 못 정하는 상태 → 사람이 골라줌. |
| **stash** | 작업 중인 변경을 잠깐 서랍에 치워두기. |
